## 开发惯例

### 日志命名

- 文件名格式: `YYYY-MM-DD.md`
- 使用 ISO 8601 日期格式
- 每天一个日志文件

### 日志内容

每篇日志应包含:
1. **今日完成**: 列出完成的工作项
2. **遇到的问题**: 记录问题和解决方案
3 **技术亮点**: 特别的技术实现
4. **代码变更**: 重要文件的修改
5. **下一步计划**: 明天的工作计划
6. **统计数据**: 行数、文件数、测试覆盖等

### 日志位置

所有日志存放在 `docs/daily-log/` 目录。

## 贡献指南

如果您是项目贡献者,请按照以下格式添加日志:

1. 创建新文件 `docs/daily-log/YYYY-MM-DD.md`
2. 使用现有日志作为模板
3. 更新本索引文件,添加新日志的链接
4. 提交 PR
