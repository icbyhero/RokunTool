## 🚀 后续开发流程

### 新UI组件开发流程

1. **规划阶段**
   - 查阅 [UI-DESIGN-SYSTEM.md](../../docs/UI-DESIGN-SYSTEM.md)
   - 确定使用的颜色模式
   - 选择合适的组件变体

2. **开发阶段**
   - 遵循暗色模式规则
   - 使用正确的颜色模式
   - 参考示例代码

3. **测试阶段**
   - 明色模式测试
   - 暗色模式测试
   - 主题切换测试
   - 对比度检查

4. **审查阶段**
   - 使用检查清单
   - 确保符合所有规范
   - 验证可访问性

### 修复现有UI问题流程

1. **审计问题**
   - 查阅 [AUDIT-FINDINGS.md](./AUDIT-FINDINGS.md)
   - 识别具体问题类型
   - 确定修复优先级

2. **应用修复**
   - 遵循颜色模式规范
   - 添加缺失的 `dark:` 变体
   - 移除硬编码颜色

3. **验证修复**
   - 测试两种主题
   - 检查对比度
   - 确认无回归
