# 微信分身插件使用指南

## 📱 功能特性

微信分身插件允许你在macOS上创建和管理多个微信实例,每个实例独立运行,互不干扰。

### 主要功能

- ✅ **一键创建分身** - 自动复制并配置微信应用
- ✅ **多实例管理** - 支持创建多个分身,独立运行
- ✅ **状态监控** - 实时显示每个分身的运行状态
- ✅ **自动签名** - 使用codesign自动签名分身应用
- ✅ **安全隔离** - 每个分身有独立的Bundle ID和数据目录
- ✅ **批量操作** - 支持批量启动、停止分身

## 🔧 系统要求

- **操作系统**: macOS 10.15+ (Catalina或更高版本)
- **微信版本**: 最新版微信(从App Store安装)
- **权限要求**:
  - 管理员权限 (用于创建分身和签名)
  - 文件系统读写权限
  - 进程管理权限

## 📥 安装和配置

### 1. 安装插件

1. 打开 RokunTool 应用
2. 进入插件市场
3. 找到"微信分身"插件
4. 点击"安装"按钮

### 2. 首次配置

首次使用时,插件会自动:

1. 检测 `/Applications/WeChat.app` 是否存在
2. 检查系统权限
3. 创建配置目录 `~/Library/Application Support/RokunTool/plugins/wechat-multi-instance`

### 3. 权限设置

插件需要以下权限,首次使用时会自动请求:

- **文件系统访问**: 读取和复制微信应用
- **进程管理**: 启动和停止微信实例
- **Shell执行**: 执行签名和系统命令
- **通知显示**: 显示操作结果提示

## 🚀 使用指南

### 创建第一个分身

1. **打开插件**
   - 在 RokunTool 中进入"微信分身"插件页面

2. **检测微信应用**
   - 插件会自动检测系统是否已安装微信
   - 如果未安装,请先从App Store安装微信

3. **创建分身**
   - 点击"创建新分身"按钮
   - 输入分身名称(例如: "工作号"、"小号")
   - 确认创建

4. **等待处理**
   - 插件会自动:
     - 复制微信应用到 `/Applications/WeChat-分身名.app`
     - 修改Bundle ID
     - 重新签名应用
     - 保存配置

5. **完成**
   - 创建成功后会显示通知
   - 分身会出现在实例列表中

### 管理分身

#### 启动分身

1. 在实例列表中找到要启动的分身
2. 点击"启动"按钮
3. 等待分身启动完成
4. 分身图标会显示为"运行中"状态

#### 停止分身

1. 在实例列表中找到要停止的分身
2. 点击"停止"按钮
3. 确认停止操作
4. 分身会安全退出

#### 删除分身

1. 在实例列表中找到要删除的分身
2. 点击"删除"按钮
3. 确认删除操作
4. 分身应用会被完全移除

### 实例状态说明

- **🟢 运行中** - 分身正在使用
- **⚪ 已停止** - 分身未运行
- **🔴 错误** - 分身配置或运行出现问题

## 🔍 常见问题解答

### Q1: 创建分身时提示"权限不足"?

**A**: 需要管理员权限。请确保:
1. 你的账户有管理员权限
2. 在弹出的权限请求对话框中点击"允许"
3. 如果仍然失败,尝试使用 `sudo` 命令手动启动应用

### Q2: 分身创建后无法启动?

**A**: 可能的原因和解决方案:
1. **签名问题** - 重新签名分身应用
2. **Bundle ID冲突** - 删除分身后重新创建
3. **微信更新** - 微信更新后需要重新创建分身
4. **系统版本不兼容** - 检查macOS版本是否满足要求

### Q3: 分身可以和原版微信同时使用吗?

**A**: 可以!每个分身都有独立的:
- Bundle ID
- 数据目录
- 进程空间
- 用户数据

它们互不干扰,可以同时登录不同的微信账号。

### Q4: 微信更新后分身还能用吗?

**A**: 微信更新后,分身可能需要重建:
1. 插件会自动检测版本不匹配
2. 点击"重建分身"按钮
3. 重新创建分身即可
4. 旧分身的聊天记录会保留

### Q5: 如何备份分身数据?

**A**: 分身数据存储位置:
```
~/Library/Containers/com.tencent.xin{分身后缀}/
~/Library/Group Containers/{群组ID}/
```

备份方法:
1. 使用Time Machine自动备份
2. 手动复制上述目录
3. 使用插件导出功能(开发中)

### Q6: 分身会影响原版微信吗?

**A**: 不会。分身是独立的应用:
- 独立的安装位置
- 独立的数据目录
- 独立的Bundle ID
- 互不影响

### Q7: 可以创建多少个分身?

**A**: 理论上没有限制,但建议:
- 一般用户: 2-3个分身
- 需求较多: 最多5个分身
- 过多分身可能影响系统性能

### Q8: 分身占用多少磁盘空间?

**A**: 每个分身约:
- 应用大小: ~200MB
- 数据大小: 根据使用情况,通常1-5GB
- 总计: 约2-5GB

## 🛠️ 故障排除指南

### 问题1: 创建分身失败

**症状**: 点击创建后提示错误

**排查步骤**:
1. 检查 `/Applications/WeChat.app` 是否存在
2. 确认磁盘空间充足(至少5GB)
3. 检查是否有管理员权限
4. 查看日志: `~/Library/Logs/RokunTool/wechat-multi-instance.log`

**解决方案**:
```bash
# 手动检查微信应用
ls -la /Applications/WeChat.app

# 检查磁盘空间
df -h

# 查看日志
cat ~/Library/Logs/RokunTool/wechat-multi-instance.log
```

### 问题2: 分身启动崩溃

**症状**: 启动分身后立即退出

**可能原因**:
1. 签名失效
2. 数据目录损坏
3. 系统兼容性问题

**解决方案**:
```bash
# 重新签名
sudo codesign --force --deep --sign - /Applications/WeChat-{分身名}.app

# 清理缓存
rm -rf ~/Library/Caches/com.tencent.xin{分身后缀}

# 重启系统
sudo reboot
```

### 问题3: 分身无法登录

**症状**: 分身启动但无法登录微信

**排查步骤**:
1. 检查网络连接
2. 确认微信账号正常
3. 尝试重启分身
4. 查看系统日志

**解决方案**:
1. 退出分身后重新登录
2. 删除分身后重新创建
3. 联系微信客服确认账号状态

### 问题4: 性能问题

**症状**: 多个分身运行时卡顿

**优化建议**:
1. 关闭不需要的分身
2. 重启电脑
3. 清理系统缓存
4. 升级硬件配置

**清理命令**:
```bash
# 清理缓存
sudo rm -rf /System/Volumes/Data/private/var/folders/*
sudo rm -rf ~/Library/Caches/*

# 重启
sudo reboot
```
